## Supabase 보정

- 최종 리뷰에서 기존 딱밤 통계 트리거가 채권자와 채무자의 누적 때림·맞음을 반대로 기록하는 결함을 발견했다.
- `correct_hit_stat_direction` 마이그레이션으로 채권자의 `hits_delivered`와 채무자의 `hits_received`가 증가하도록 교정하고 기존 합계를 방향성 의무에서 다시 계산했다.
- 원격 프로젝트 적용 후 함수 방향과 프로필 합계 불일치 0건을 SQL로 확인했으며, Supabase 보안 Advisor는 경고 0건이다.
- 알려진 구조적 위험: 현재 방 상태 JSON은 방 멤버가 갱신하는 신뢰 기반 구조라 악성 클라이언트의 상태 위조를 완전히 막지는 못한다. 서버 권위형 전환 RPC는 별도 보안 강화 작업이 필요하다.
