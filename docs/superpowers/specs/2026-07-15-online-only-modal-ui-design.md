# 온라인 전용 모달 UI 설계

## 목표

딱밤소사이어티를 로컬 AI 데모가 없는 온라인 전용 제품으로 정리한다. 로그인과 계정별 딱밤 장부는 게임판에서 분리된 페이지 내부 모달로 제공하고, 게임 중에는 자기 패 이름과 간결한 족보 롤업을 확인할 수 있게 한다.

## 확정 범위

- 로컬 가상 계정, AI 턴, 로컬 게임 테이블, 로컬 메모리 장부를 제거한다.
- Supabase 인증, 온라인 방, Realtime 게임, 계정별 딱밤 의무 데이터는 유지한다.
- 로그인/회원가입을 페이지 내부 중앙 모달로 분리한다.
- 계정별 딱밤 장부를 별도 모달로 분리한다. 모바일에서는 바텀시트 형태로 표시한다.
- 자기 패에는 `내 패 · {패 이름}`과 `족보 보기`만 표시한다.
- 족보 롤업은 숫자 패를 축약하고 외우기 어려운 항목을 자세히 표시한다.

## 비범위

- 별도 브라우저 창을 `window.open()`으로 띄우지 않는다.
- 패 조합의 생성 원리나 월 조합 설명은 제공하지 않는다.
- 베팅 규칙, 딱밤 정산 규칙, Supabase 스키마, Realtime 저장 형식은 변경하지 않는다.
- 상대 패는 쇼다운 전에 설명하거나 공개하지 않는다.

## 화면 구조

### 공통 헤더

- 브랜드와 `온라인 접속` 상태를 표시한다.
- 비로그인 상태에는 `로그인` 버튼을 표시한다.
- 로그인 상태에는 계정 이름, `딱밤 장부 · N건`, `로그아웃`을 표시한다.
- 장부 건수는 현재 계정과 연관된 방향성 의무 수를 사용한다.

### 비로그인 상태

- 온라인 전용 제품 소개와 비활성 로비 미리보기를 보여준다.
- 주 행동은 `로그인하고 시작` 하나로 제한한다.
- 버튼을 누르면 인증 모달이 열린다.

### 로그인 후 로비

- 새 방 생성과 6자리 코드 입장을 같은 온라인 허브에서 제공한다.
- 참가 인원은 2~4명이다.
- 방 참가 후에는 방 코드, 좌석, 준비 상태, 준비/시작/나가기 행동을 표시한다.

### 게임 중

- `OnlineRoomGame`이 페이지의 주 콘텐츠가 된다.
- 자기 패는 베팅 중에도 앞면으로 보이고, 상대 패는 쇼다운 전까지 뒷면이다.
- 기존 `받기`, `올리기`, 다음 판 흐름은 유지한다.

## 로그인 모달

- 네이티브 `<dialog>`와 `showModal()`을 사용한다.
- 제목은 `딱밤 계정으로 입장`이다.
- 로그인/회원가입 탭과 기존 이메일·비밀번호·표시 이름 입력을 재사용한다.
- 인증 오류는 모달 안에서 `role="alert"`로 표시한다.
- 로그인 성공 시 모달을 닫고 온라인 로비를 활성화한다.
- 닫기 버튼, `Esc`, 배경 클릭으로 닫을 수 있다.
- 닫은 뒤 초점은 모달을 열었던 버튼으로 돌아간다.

## 딱밤 장부 모달

- 헤더의 `딱밤 장부 · N건` 버튼으로 연다.
- 제목은 `내 딱밤 장부`이다.
- `내가 때릴 딱밤`, `내가 맞을 딱밤`, `누적 때림`, `누적 맞음`을 요약한다.
- 각 의무는 채무자 → 채권자 방향을 그대로 보여주며 반대 방향 의무와 상계하지 않는다.
- 현재 계정이 채무자인 항목에만 `한 대 때림` 버튼을 제공한다.
- 기록 성공은 `aria-live="polite"`, 실패는 모달 내부 오류로 알린다.
- 데스크톱은 중앙 모달, 좁은 화면은 최대 높이 `88dvh`의 하단 바텀시트로 표시한다.

## 내 패와 족보 롤업

- `round.hands[userId]`를 기존 `evaluateHand()`에 전달해 자기 패 이름을 계산한다.
- 접힌 상태에는 `내 패 · {evaluation.name}`과 `족보 보기`만 표시한다.
- `특수패 1위`처럼 별도의 순위 문장은 표시하지 않는다.
- 펼친 상태에는 다음 순서만 표시한다.
  - 광땡: `38광땡 〉 18광땡 〉 13광땡`
  - 땡: `10땡 〉 … 〉 1땡`
  - 특수패: `알리 〉 독사 〉 구삥 〉 장삥 〉 장사 〉 세륙`
  - 끗: `갑오 〉 8끗 〉 … 〉 1끗 〉 망통`
- 현재 패 이름이 목록에 있으면 해당 텍스트만 강조한다.
- 롤업은 자기 좌석 내부에서 열고 닫으며, 상대 좌석에는 쇼다운 전 표시하지 않는다.

## 컴포넌트 경계

- `src/app/page.tsx`: 온라인 전용 루트만 렌더링한다.
- `src/components/account-room-panel.tsx`: 인증 세션, 방/멤버, 장부와 Realtime 데이터의 소유권을 유지하고 온라인 셸을 구성한다.
- `src/components/account-auth-dialog.tsx`: 인증 모달의 UI와 포커스/닫기 동작을 담당한다.
- `src/components/hit-ledger-dialog.tsx`: 장부 모달의 표시와 기록 행동을 담당한다.
- `src/components/online-room-game.tsx`: 자기 패 이름과 족보 롤업을 추가한다.
- `src/components/game-table.tsx`: 참조를 제거한 뒤 삭제한다.
- `src/app/globals.css`: 로컬 테이블/장부 스타일을 삭제하고 온라인 셸·모달 스타일을 정리한다.

## 데이터 흐름

1. `AccountRoomPanel`이 Supabase 세션을 구독한다.
2. 비로그인 사용자가 인증 모달에서 로그인하면 기존 auth listener가 사용자와 프로필을 다시 불러온다.
3. 방 생성/입장/준비/게임은 기존 Supabase 쿼리와 Realtime 구독을 그대로 사용한다.
4. `OnlineRoomGame`은 현재 라운드에서 자기 패를 평가해 표시만 추가한다.
5. 장부 모달은 부모가 소유한 `hit_obligations`를 받고, 한 대 기록 후 기존 `refreshLedger()`로 갱신한다.

## 오류 처리

- Supabase 설정이 없으면 온라인 서비스를 사용할 수 없다는 기존 안내를 유지한다.
- 인증 오류와 방/장부 오류를 분리해 잘못된 화면에 메시지가 남지 않게 한다.
- Realtime 버전 충돌은 기존 최신 상태 재동기화 동작을 유지한다.
- 모달이 열린 동안 배경 스크롤을 막고, 닫힐 때 복원한다.

## 검증

- 기존 게임 엔진 및 방 유닛 테스트를 모두 통과해야 한다.
- 로그인 모달의 열기/닫기, 탭 전환, `Esc`, 초점 복귀를 검증한다.
- 장부 모달의 열기/닫기와 `한 대 때림` 가능 조건을 검증한다.
- 자기 좌석에만 베팅 중 패 이름과 족보 롤업이 표시되는지 확인한다.
- 족보 롤업의 네 그룹과 축약 표기가 확정 문구와 일치하는지 확인한다.
- 데스크톱과 모바일 화면에서 모달이 게임 조작을 방해하지 않는지 시각 검증한다.
- lint, TypeScript, 테스트, 프로덕션 빌드 후 라이브 Vercel 화면의 콘솔 오류를 확인한다.

## 알려진 위험

- 온라인 라운드 JSON에 모든 패가 포함된 기존 구조는 유지되므로 기술 사용자가 개발자 도구로 상대 패를 볼 수 있는 위험이 남는다.
- 모달 분리 과정에서 기존 `busy`와 `notice` 상태를 그대로 공유하면 메시지 위치가 섞일 수 있으므로 인증·장부 오류 상태를 분리해야 한다.
