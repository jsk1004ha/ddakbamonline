# 오프라인 딱밤 빚 등록 설계

## 목표

회원가입 시 사용자가 자신의 이름을 필수로 입력하게 하고, 로그인한 사용자가 딱밤 장부 창에서 가입된 상대방을 이름으로 찾아 오프라인 게임에서 생긴 딱밤 빚을 즉시 등록할 수 있게 한다. 등록된 빚은 게임에서 발생한 빚과 같은 방식으로 양쪽 계정 장부에 표시되고, 기존 `한 대 때림` 계수 기능으로 소진된다.

## 확정된 제품 규칙

- 회원가입 입력 명칭은 `닉네임`이 아니라 `이름`이다.
- 이름은 2~24자이며 회원가입 시 필수다.
- 오프라인 빚의 상대방은 가입된 계정만 선택할 수 있다.
- 로그인한 사용자는 반드시 채권자 또는 채무자 중 한 명이어야 한다.
- 등록 시 상대방 승인을 기다리지 않고 양쪽 계정 장부에 즉시 반영한다.
- 사용자는 `내가 때릴 딱밤` 또는 `내가 맞을 딱밤` 방향을 명시한다.
- 횟수는 1 이상의 정수이며 상한을 두지 않는다.
- 서로 반대 방향의 빚은 상계하지 않고 각각 남긴다.
- 동명이인은 이름을 주 정보로, 계정 아이디를 보조 정보로 표시해 구분한다.

## 데이터 모델

기존 `public.hit_obligations`를 게임·오프라인 공용 장부로 확장한다.

- `game_result_id`를 nullable로 변경한다.
- `source` 열을 추가하며 값은 `game` 또는 `offline`이다.
- `created_by` 열을 추가해 오프라인 빚을 등록한 계정을 기록한다.
- 게임 빚은 `source = 'game'`, `game_result_id is not null`, `created_by is null`이다.
- 오프라인 빚은 `source = 'offline'`, `game_result_id is null`, `created_by is not null`이다.
- 기존 레코드는 모두 `game`으로 보정한다.
- 게임 횟수·승률 통계는 `game_results`에만 연결되므로 오프라인 빚 등록으로 변하지 않는다.

별도 오프라인 장부 테이블은 만들지 않는다. 조회·실시간 반영·타격 계수 로직을 기존 장부와 공유해 중복을 피한다.

## 서버 권한 경계

브라우저의 직접 INSERT 권한은 계속 금지한다. 새 인증 RPC `public.add_offline_hit_obligation`만 오프라인 빚을 생성한다.

RPC 입력:

- `counterparty_id uuid`: 선택한 상대 계정
- `direction text`: `i_hit` 또는 `i_owe`
- `hits text`: 임의 정밀도의 양의 정수 문자열

RPC 검증:

- 로그인 세션이 존재한다.
- 상대 계정이 실제 프로필로 존재한다.
- 상대방은 현재 사용자와 다르다.
- 방향 값이 허용 목록에 포함된다.
- 횟수가 앞자리 0이 없는 1 이상의 정수다.
- 현재 사용자를 방향에 맞춰 채권자 또는 채무자로 강제한다.

함수는 `SECURITY DEFINER`, 빈 `search_path`, 명시적 객체 경로, 공개·익명 실행권한 회수, `authenticated` 전용 실행권한을 사용한다. 사용자 메타데이터는 권한 결정에 사용하지 않는다.

## 사용자 인터페이스

### 회원가입

- 회원가입 탭에서 `이름` 필드를 아이디보다 먼저 표시한다.
- 공백 제거 후 2~24자를 검증하고 기존 `display_name` 메타데이터로 전달한다.
- 로그인 탭에서는 이름 필드를 표시하지 않는다.

### 딱밤 장부

장부 요약 아래에 `오프라인 빚 추가` 폼을 둔다.

1. 방향 선택: `내가 때릴 딱밤` / `내가 맞을 딱밤`
2. 상대 이름 검색
3. 검색 결과에서 가입 계정 선택
4. 딱밤 횟수 입력
5. `장부에 추가` 실행

검색 결과는 이름을 크게, 계정 아이디를 작게 표시한다. 선택 전에는 제출할 수 없다. 성공하면 폼을 초기화하고 장부를 새로 불러오며 성공 안내를 표시한다.

각 장부 항목에는 `게임` 또는 `오프라인` 출처 배지를 표시한다. 기존 방향 문장과 남은 횟수, 완료 횟수, `한 대 때림` 동작은 유지한다.

## 오류 처리

- 검색어가 없거나 너무 짧으면 검색하지 않는다.
- 상대가 존재하지 않거나 삭제된 경우 다시 선택하도록 안내한다.
- 자기 자신 선택, 잘못된 방향, 0·음수·소수·비정규 정수는 서버와 클라이언트 모두 거부한다.
- RPC 실패 시 입력값을 유지하고 장부 창 안에 오류를 표시한다.
- 실시간 이벤트 또는 RPC 성공 후 현재 계정 장부를 다시 조회해 양쪽 기기의 표시를 맞춘다.

## 검증

- 회원가입 이름 필수·공백 제거·길이 검증 테스트
- 마이그레이션의 열, 제약조건, 권한 회수, RPC 인증·당사자·정수 검증 테스트
- 브라우저가 직접 장부 INSERT를 하지 않고 RPC만 호출하는 회귀 테스트
- 이름 검색 결과와 동명이인 보조 아이디 표시 테스트
- 방향별 채권자·채무자 매핑 테스트
- 오프라인 빚이 게임 통계를 바꾸지 않는 운영 SQL 검증
- 등록 직후 양쪽 장부 노출과 `한 대 때림` 감소 동작 브라우저 검증
- 전체 테스트, ESLint, TypeScript, Next.js 프로덕션 빌드

## 범위 밖

- 상대방 승인 흐름
- 비회원 자유 텍스트 장부
- 오프라인 빚 수정·삭제
- 동일 기기 다중 계정 또는 담합 탐지
